### Sobre

Engenheiro full-stack focado em **JavaScript / TypeScript**, com especialidade em produtos construídos sobre **Next.js + Supabase**. Gosto do caminho completo: arquitetar o modelo de dados, modelar RLS, desenhar a UI com RSC e levar o app até o deploy com confiança. Busco vagas onde o front e o back conversam de igual pra igual.

### Projeto em destaque: Moneda

<p align="left">
  <a href="https://github.com/AdvenTureee/moneda"><img alt="Moneda" src="https://img.shields.io/badge/Moneda-despesas_pessoais_com_IA-4A5568?style=for-the-badge&logo=vercel&logoColor=white&labelColor=2D3748" /></a>
  &nbsp;
  <img alt="Status" src="https://img.shields.io/badge/status-MVP_1_em_produção-2F855A?style=for-the-badge&labelColor=2D3748" />
</p>

> App de controle de gastos com IA que entende linguagem natural em português ("almoço 35 ifood") e devolve resumos que **explicam** o dinheiro, não só mostram gráficos. Pensado para o brasileiro que abandona planilha e app de finanças em duas semanas.

**O que aprendi levando do zero ao deploy:**

- **Arquitetura Next.js 16 (App Router, RSC).** Separação server/client, streaming de dados, route handlers para IA e exportações.
- **Modelagem de dados no Supabase.** Postgres, Row Level Security por usuário, cookies SSR, migrations versionadas.
- **Integração de LLMs na prática.** Groq (`llama-3.1-8b-instant`, `llama-3.3-70b-versatile`) para categorização de texto livre e insights.
- **PWA de verdade.** Service worker, manifest, instalável, tema claro/escuro, mobile-first.
- **Disciplina de produto.** Escrever PRODUCT.md com personas e KPIs antes do código, e DESIGN.md antes do componente.
- **Segurança como hábito.** Auditoria de segredos, mascaramento de PII, rate limit nas rotas de IA.

### Princípios que defense no código

1. **Produto antes de tecnologia.** A feature só vale se resolver uma dor mensurável.
2. **RSC quando possível, `"use client"` quando necessário.** Não carregar JS à toa.
3. **Segurança no modelo, não no overlay.** RLS no banco; secrets fora do repo.
4. **Design system escrevendo código.** Tokens primeiro, componentes depois.
